## 2. 接口路由

| 目标 | 类型 | 选定接口 | 为什么这样选 | 负责人 |
|------|------|----------|--------------|--------|
| 示例：GitHub | 托管服务 | `gh` | 已有成熟官方 CLI | 工程 |
| 示例：Figma | 设计工具 | Figma bridge | 比自造 CLI 更稳定 | 设计 |
| 示例：内部管理后台 | 网站 | `opencli` | 需要浏览器驱动动作 | 运营 |
| 示例：有源码的桌面工具 | 应用 / 仓库 | `CLI-Anything` | 没有稳定 CLI，但源码可用 | 工程 |

## 5. 执行计划

1. 先确认访问权限和鉴权状态。
2. 给每个目标确定接口路径。
3. 写出或补齐最小命令集。
4. 跑通目标工作流。
5. 记录验证方式和失败回退路径。

## 7. 备注

- 优先官方 CLI，不要先生成适配器。
- 网站、Electron、外部工具优先考虑 `opencli`。
- 只有在目标没有成熟 CLI 且源码可用时，才考虑 `CLI-Anything`。
- 记录真正可运行的命令，不要只写抽象意图。
