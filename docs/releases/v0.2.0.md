# Release v0.2.0

这是 `agent-delivery-os` 的第一次“可初始化、可验收”版本。

## 这版做了什么

这一版把仓库从“模板和自动化母版”推进成了“新仓库可一键起步并可自检”的交付系统。

## 这版包含什么

- 本地初始化：
  - `pnpm bootstrap:repo`
- GitHub 初始化：
  - `pnpm bootstrap:github`
- 一体化初始化：
  - `pnpm bootstrap:all`
- 仓库健康检查：
  - `pnpm doctor`

## 这意味着什么

现在一个新的业务仓库从 template 创建后，可以做到：

- 先生成 `docs/project/` 项目骨架
- 再配置 GitHub description、homepage、topics 和 labels
- 最后用 `doctor` 检查本地结构和远端协作面是否已经接好

## 推荐起步路径

```bash
pnpm bootstrap:all -- --project "新项目名" --repo-name "new-product-repo" --repo <owner/repo>
pnpm doctor -- --repo <owner/repo>
```

## 当前状态

到这一版为止，仓库已经具备：

- 三模式交付工作流
- issue 到 workspace 的自动推进
- PR guardrails 和 merge 后闭环
- 模板仓库到业务仓库的一键初始化
- 本地和远端的仓库健康检查

## 下一步方向

- 增加更细的 business repo doctor 检查项
- 增加从 README 或项目代号自动回写仓库文案的能力
- 补充更多真实业务仓库落地样本
