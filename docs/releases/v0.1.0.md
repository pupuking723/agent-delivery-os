# Release v0.1.0

这是 `agent-delivery-os` 的第一版公开发布。

## 这版是什么

一套面向 agent 的交付操作系统母版仓库，目标是把：

`idea -> spec -> implementation -> validation -> release -> feedback`

变成可复用、可协作、可自动推进的仓库工作流。

## 这版包含什么

- 三模式工作流：
  - `Flash Launch`
  - `Product Iteration`
  - `Legacy Maintenance`
- 文档体系：
  - `docs/delivery-os/`
  - `docs/workflows/`
  - `docs/templates/`
  - `docs/templates-zh/`
  - `docs/prompt-pack/`
- 执行脚本：
  - `scripts/delivery-os/init.mjs`
  - `scripts/delivery-os/from-issue.mjs`
  - `scripts/github/sync-labels.mjs`
- GitHub 自动化：
  - issue 模板
  - labels
  - kickoff comment
  - artifact checklist
  - PR guardrails
  - merge 后 checklist 勾选
  - merge 后 `iteration-log` 提醒
- 示例与教学材料：
  - `docs/playbook.md`
  - `examples/workspace-product-iteration/`
  - `examples/demo-product-iteration/`

## 当前适合谁

- 想把 Claude Code / agent 工作流产品化的人
- 想把需求、文档、开发、发布收敛到同一条仓库路径的人
- 想给多项目建立统一交付操作系统的人

## 下一步方向

- 接入真实业务仓库
- 在更复杂的项目里验证多团队协作路径
- 继续增强 release 和 feedback 自动化
