# Contributing

## 目标

这个仓库不是普通模板仓库，而是一个持续演进的交付系统。

提交修改前，先判断你是在改：

- 模式路由
- 文档模板
- Prompt 包
- GitHub 协作入口
- 自动化脚本

## 默认流程

1. 先确认改动目标属于哪一层
2. 如果改动影响流程规则，先更新 `docs/delivery-os/`
3. 如果改动影响执行入口，同步更新 `CLAUDE.md`
4. 如果改动影响仓库协作入口，同步更新 `.github/`
5. 如果改动影响自动生成逻辑，同步更新 `scripts/delivery-os/init.mjs`
6. 如果改动影响标签或协作约束，同步更新 `.github/labels.json` 和 `docs/repo/`
7. 最后更新 `README.md` 或相关说明文档

## 变更原则

- 不要只加模板，不补入口
- 不要只改规则，不改说明
- 不要让脚本行为和文档矩阵冲突
- 新增模式前，优先检查是否可以复用现有三模式
- 不要只改 GitHub 远端标签，不回写本仓库配置

## 提交建议

- 用 issue 模板发起变更
- PR 必须说明模式、影响范围、验证方式
- 对于流程级改动，至少给一个真实使用场景
