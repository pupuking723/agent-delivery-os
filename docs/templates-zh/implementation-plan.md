## 6. 开发 Prompt

### 6.1 编码 Prompt

```text
请根据已批准的 PRD 和设计规格实现本次变更。
范围：
- [列出具体文件或模块]
要求：
- [列出关键验收标准]
约束：
- 除非必要，不要破坏现有架构
- 保持变更最小且易审查
- 为行为变更补充或更新测试
验证：
- 优先运行最小但可靠的测试集合
- 明确说明哪些路径未验证
输出：
- 变更摘要
- 影响文件
- 已运行测试
- 剩余风险
```

### 6.2 评审 Prompt

```text
请对照 PRD、设计规格和实现计划审查这次改动。
重点关注：

- 边界情况
- 缺失测试
- 设计不一致
- 发布风险
先按严重级别输出问题，并附文件引用。
```

### 6.3 QA Prompt

```text
请根据 PRD 和发布清单验证这个功能。

- 无障碍抽查
- 回归风险
输出：
- 各场景通过或失败

- 验证缺口
```
