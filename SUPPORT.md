# Support

## Best Way To Get Help

Use GitHub Issues for:

- setup problems
- workflow questions
- template usage questions
- bug reports
- feature requests

Choose the issue form that matches the operating mode when possible:

- `Flash Launch`
- `Product Iteration`
- `Legacy Maintenance`

## Before Opening An Issue

- read [README.md](/Users/mac/Desktop/other/全栈/README.md)
- read [docs/playbook.md](/Users/mac/Desktop/other/全栈/docs/playbook.md)
- check [docs/repo/github-setup.md](/Users/mac/Desktop/other/全栈/docs/repo/github-setup.md)
- confirm whether the question is about this repository or a downstream product repository

## What To Include

- your goal
- current mode
- relevant repository path or workspace path
- the command you ran
- expected result
- actual result

## What This Repository Does Not Provide

- guaranteed SLA
- private implementation consulting
- support for unrelated third-party tools outside this repo's documented workflows

## Security Issues

Do not file public issues for security vulnerabilities.
Use [SECURITY.md](/Users/mac/Desktop/other/全栈/SECURITY.md) instead.
